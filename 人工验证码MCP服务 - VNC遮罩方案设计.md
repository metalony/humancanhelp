# 人工验证码MCP服务 - 设计文档

***

## 一、核心定位

**一句话**: AI的验证码外包平台

做成MCP服务，让所有AI（Claude/GPT/Cursor等）都能调用真人完成验证任务。

***

## 二、核心约束

| 约束            | 说明                          |
| ------------- | --------------------------- |
| **派发方多样化**    | Windows/Mac/Linux + 网页/桌面软件 |
| **作业员只Web操作** | 不接触AI环境，通过Web界面完成任务         |
| **隐私可控**      | 派发方可指定遮罩区域，作业员看不到敏感信息       |

***

## 三、技术架构

### 3.1 整体架构

```plain&#x20;text
派发方                          服务端                    作业员
┌──────────────┐         ┌──────────────────┐         ┌──────────────┐
│              │         │                  │         │              │
│  全屏VNC     │──原始流─→│  VNC代理服务     │──遮罩流─→│  noVNC Web   │
│  (密码保护)  │         │  ├─接收原始流    │         │  只看到遮罩后 │
│              │←──操作──│  ├─实时遮罩引擎  │←──操作──│              │
│              │         │  └─权限控制      │         │              │
└──────────────┘         └──────────────────┘         └──────────────┘
                               ▲
                               │ 遮罩配置
                               │
                    ┌──────────────────┐
                    │   MCP Server     │
                    │   接收遮罩参数   │
                    └──────────────────┘
```

### 3.2 数据流

```plain&#x20;text
1. AI请求验证码帮助
   MCP Client → MCP Server: create_task(遮罩配置)
   
2. MCP Client启动VNC Server
   生成临时密码 + 启动VNC Server（全屏）
   
3. VNC代理启动遮罩引擎
   连接派发方VNC → 接收原始流 → 实时遮罩 → 输出给作业员

4. 任务派发给作业员
   MCP Server → 作业员Web: push_task(vnc_url, instruction)

5. 作业员连接noVNC
   看到遮罩后的界面 → 操作（点击/拖拽/输入）
   
6. 操作经过代理过滤
   只允许在allow_regions内操作 → 其他区域操作被拦截
   
7. 任务完成/超时
   MCP Server → MCP Client: task_completed
   关闭VNC连接 → 销毁会话
```

## 四、遮罩方案设计

### 4.1 遮罩在代理层完成

**关键优势**:

* 作业员**永远看不到**被遮罩的内容

* 派发方完全控制遮罩策略

* 不需要各OS的特殊配置

### 4.2 遮罩模式

| 模式           | 效果   | 适用场景         |
| ------------ | ---- | ------------ |
| **black**    | 纯黑遮罩 | 高敏感信息（账号、密码） |
| **blur**     | 高斯模糊 | 中敏感信息（文档内容）  |
| **pixelate** | 马赛克  | 低敏感信息（工具栏）   |

### 4.3 区域配置示例

```json
{
  "mask_regions": [
    {"x": 0, "y": 0, "w": 1920, "h": 100},
    {"x": 0, "y": 200, "w": 300, "h": 800}
  ],
  "mask_mode": "blur",
  "allow_regions": [
    {"x": 500, "y": 400, "w": 400, "h": 300}
  ]
}
```

## 五、操作权限控制

### 5.1 鼠标操作过滤

只允许在 `allow_regions` 内操作，其他区域点击被拦截。

### 5.2 键盘操作过滤

禁止按键列表：

* Win / Meta\_L / Meta\_R（Windows键）

* Alt+Tab（切换窗口）

* Ctrl+Alt+Del（系统操作）

***

## 六、各OS VNC支持

| OS      | VNC Server | 配置方式                 |
| ------- | ---------- | -------------------- |
| Windows | UltraVNC   | `winvnc.exe` 全屏共享    |
| macOS   | 内置VNC      | 系统偏好设置 → 屏幕共享        |
| Linux   | x11vnc     | `x11vnc -display :0` |

**统一方案**: 全屏VNC + 代理层遮罩，不需要窗口级配置

## 七、MCP接口设计

### 7.1 start\_human\_session

**输入参数**:

| 参数                       | 类型     | 说明                  |
| ------------------------ | ------ | ------------------- |
| vnc.port                 | number | 本地VNC端口（默认5900）     |
| vnc.password             | string | "auto"自动生成临时密码      |
| privacy.mask\_regions    | array  | 遮罩区域列表              |
| privacy.mask\_mode       | string | black/blur/pixelate |
| privacy.allow\_regions   | array  | 允许操作区域              |
| permission.blocked\_keys | array  | 禁止按键列表              |
| task.instruction         | string | 操作提示                |
| task.timeout             | number | 超时秒数                |

**输出**:

| 参数          | 说明       |
| ----------- | -------- |
| session\_id | 会话ID     |
| vnc\_url    | 作业员连接URL |
| password    | 临时密码     |

### 7.2 end\_human\_session

**输入**: session\_id

**输出**: 状态、耗时、操作次数

***

## 九、安全机制

### 9.1 派发方安全

* 临时密码：每次会话随机生成

* 遮罩隔离：作业员看不到敏感信息

* 区域限制：只能在允许区域操作

* 时间限制：超时自动断开

### 9.2 作业员管理

* 手机号实名认证

* 技能测试（准确率>95%）

* 保证金制度

* 评分体系

***

## 十、场景示例

### 场景1: 浏览器验证码

遮罩：地址栏 + 书签栏 + 侧边栏
允许：验证码区域（600, 400, 400×300）

### 场景2: 桌面软件登录

遮罩（black模式）：账号输入框 + 密码输入框
允许：验证码区域
